# AI Vendor Review Process

Use this lightweight process for sample portfolio reviews.

## 1. Intake

Document the product name, business owner, use case, users, and data types.

## 2. Data Classification

Identify whether the product will process public, internal, confidential, regulated, or security-sensitive data.

## 3. Questionnaire

Send the vendor questionnaire and request supporting documentation for IAM, logging, data protection, AI training, subprocessors, and incident response.

## 4. Rule-Based Assessment

Run the sample vendor JSON profile through `vendor_risk_assessment.py` to produce a draft risk report.

## 5. Security Review

Review missing controls, AI-specific risks, data protection concerns, IAM concerns, and logging concerns.

## 6. Decision

Document whether the vendor is approved, conditionally approved, or not approved. Record required security conditions.

## 7. Reassessment

Reassess at least annually or when the vendor changes data use, model training, subprocessors, authentication, or logging capabilities.
