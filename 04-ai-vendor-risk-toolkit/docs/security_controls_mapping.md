# Security Controls Mapping

This mapping helps connect vendor questionnaire answers to security control areas.

| Control Area | Vendor Evidence to Request | Why It Matters |
| --- | --- | --- |
| IAM | SSO, MFA, SCIM, RBAC documentation | Reduces account takeover and excessive access risk. |
| Logging | Audit log samples and export options | Supports SOC monitoring and investigations. |
| Data Protection | Encryption, retention, deletion, residency documentation | Protects customer data lifecycle. |
| AI Training | Model training and product improvement policy | Clarifies whether customer data may train models. |
| Admin Controls | Tenant controls, integration controls, export controls | Supports least privilege and governance. |
| Incident Response | Security notification SLA and escalation contacts | Defines expectations during vendor incidents. |
| Third-Party Risk | Subprocessor list and change notification process | Identifies downstream data exposure. |
| Compliance | SOC 2, ISO 27001, privacy documentation | Supports governance and assurance review. |
